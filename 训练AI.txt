# 二、构建交易环境（Trading Environment）

环境模拟：

```

订单执行
手续费
滑点
```

很多团队会使用 **OpenAI Gym 风格环境**。

环境接口：

```python
class TradingEnv:

    def reset(self):
        pass

    def step(self, action):
        pass
```

`reset()`：

```
开始新的交易周期
```

`step()`：

```
执行交易
返回奖励
```

# 三、状态空间（State）

常见 State 包括：


例如：

```python

    price,

    orderbook_imbalance

```


例如：

```

```

# 四、动作空间（Action）

Action 是模型可以做的决策。

常见设计：

### 离散动作

```
0 = Hold
1 = Buy
2 = Sell
```

示例：

```python
action_space = 3
```

---

### 连续动作

更高级的设计：

```
-1 ~ 1
```

表示：

```
仓位比例
```

例如：

```
-1 = 全空
0 = 空仓
1 = 全多
```

---

# 五、奖励函数（Reward）

Reward 是强化学习最关键部分。

简单奖励：

```
reward = 当前资产变化
```

示例：

```python
reward = portfolio_value_t - portfolio_value_t_minus_1
```

---

### 更真实的 Reward

加入交易成本：

```
reward = pnl - fee - slippage
```

示例：

```python
reward = pnl - trading_cost
```

---

### 风险调整奖励

例如：

```
Sharpe Ratio
```

奖励：

```
reward = return / volatility
```

这样可以：

```
避免高风险策略
```

# 四、Alpha Generator（自动 Alpha 生成）

Alpha Generator 会自动组合特征。

例如：

基础特征：

```text
price
volume
volatility
```

系统组合生成：

```text
price / volume
volume * volatility
log(price) * momentum
```

生成方法：

```python
for f1 in features:
    for f2 in features:
        new_alpha = f1 / f2
```

最终可能生成：

```text
数万个候选 Alpha
```

# 一、什么是 Alpha

Alpha 本质上就是：

```text
可预测未来收益的信号
```

例如：

```text
动量因子
均值回归
资金费率
订单簿 imbalance
```

数学表示：

```text
alpha(t) → 预测 future_return(t+1)
```

如果：

```text
corr(alpha, future_return) > 0
```

说明该 Alpha 有预测能力。

---

# 二、单个 Alpha 的问题

单个策略通常不稳定：

```text
Sharpe = 1 ~ 2
```

但可能经历：

```text
长期失效
回撤
市场变化
```

因此机构通常运行：

```text
100 ~ 10000 个 Alpha
```

# 三、Momentum 因子（动量）

动量是最经典的因子。

核心思想：

```
上涨的资产继续上涨
下跌的资产继续下跌
```

### 1 简单动量

```
momentum = price / price_lookback - 1
```

Python：


### 2 多周期动量

```

```

```python
mom5 = close.pct_change(5)

```

# 四、Mean Reversion 因子（均值回归）

核心思想：

```
价格偏离均值后会回归
```

### 1 Z-score 因子

```
z = (price - mean) / std
```

Python：

```python

```

交易逻辑：

```
z > 2   做空
z < -2  做多
```

---

### 2 Bollinger Band

```
upper = mean + 2*std
lower = mean - 2*std
```

策略：

```
触及上轨做空
触及下轨做多
```

# 四、TWAP（Time Weighted Average Price）

TWAP 是最简单的执行算法。

思想：

```text
把订单平均分配到时间上
```

例如：

```text
100 BTC
1 小时完成
```

拆分：

```text
每分钟买入
1.67 BTC
```

示例代码：

```python
order_size = 100
intervals = 60

slice_size = order_size / intervals
```

优点：

```text
简单
稳定
```

缺点：

```text
忽略市场流动性
```

---

# 五、VWAP（Volume Weighted Average Price）

VWAP 根据 **市场成交量** 调整交易速度。

公式：

```text
VWAP = Σ(price × volume) / Σ(volume)
```

执行策略：

```text
成交量大 → 多交易
成交量小 → 少交易
```

示例：

```python
target_volume = market_volume * participation_rate
```

优点：

```text
更贴近市场交易节奏
```

---

# 六、POV（Participation of Volume）

POV 表示：

```text
参与市场成交量的比例
```

例如：

```text
POV = 5%
```

意味着：

```text
市场成交 100 BTC
你交易 5 BTC
```

示例代码：

```python
order_size = market_volume * 0.05
```

优点：

```text
自动适应市场流动性
```
