# RTX 3090 优化配置指南

## 概述

本文档说明如何针对 RTX 3090 24GB VRAM 优化 Qwen3.5-7B 微调参数。

## RTX 3090 规格

| 规格 | 数值 |
|------|------|
| **VRAM** | 24GB GDDR6X |
| **CUDA 核心** | 10496 |
| **Tensor 核心** | 328 |
| **显存带宽** | 936GB/s |
| **功耗** | 350W |

## 参数调整说明

### 1. 通用配置（A100）vs RTX 3090

| 参数 | A100 40GB | RTX 3090 24GB | 调整原因 |
|------|-----------|--------------|---------|
| **max_seq_length** | 2048 | **1536** | 减少显存占用 |
| **lora_r** | 64 | **32** | 减少可训练参数量 |
| **lora_alpha** | 128 | **64** | 配合 lora_r 调整 |
| **batch_size** | 4 | **2** | 减半适应显存 |
| **gradient_accumulation** | 8 | **16** | 补偿 batch size |
| **learning_rate** | 1e-4 | **8e-5** | 稍低的学习率更稳定 |
| **max_steps** | 1000 | **1500** | 增加训练步数 |
| **warmup_steps** | 100 | **150** | 相应增加预热步数 |

### 2. 详细参数解释

#### max_seq_length: 2048 → 1536
- **原因**: RTX 3090 显存比 A100 少 40%
- **效果**: 减少约 25% 的显存占用
- **影响**: 历史 K 线从 100 根减少到约 75 根，仍然足够

#### lora_r: 64 → 32
- **原因**: 降低 LoRA 矩阵维度
- **效果**: 可训练参数从约 160M 减少到约 80M
- **影响**: 略微降低模型容量，但训练速度更快

#### lora_alpha: 128 → 64
- **原因**: 保持 lora_alpha = 2 × lora_r 的比例
- **效果**: 保持 LoRA 缩放比例不变
- **影响**: 无明显负面影响

#### batch_size: 4 → 2
- **原因**: RTX 3090 显存不足以支撑 batch_size=4
- **效果**: 单批显存占用减半
- **影响**: 梯度累积步数加倍补偿

#### gradient_accumulation: 8 → 16
- **原因**: 补偿 batch_size 的减少
- **效果**: 有效 batch size = 2 × 16 = 32（与原来一致）
- **影响**: 训练稳定性保持不变

#### learning_rate: 1e-4 → 8e-5
- **原因**: 小 batch size 下稍低的学习率更稳定
- **效果**: 减少训练波动
- **影响**: 收敛速度稍慢，但更稳定

#### max_steps: 1000 → 1500
- **原因**: 小 batch size + 低学习率需要更多步数
- **效果**: 保持总训练样本量一致
- **影响**: 训练时间增加约 50%

#### warmup_steps: 100 → 150
- **原因**: 按比例增加预热步数
- **效果**: 保持预热比例
- **影响**: 无负面影响

## 使用方法

### 第一步: 上传文件

将以下文件上传到 RTX 3090 实例:

```
1. qwen_trading_samples_20260320_145112.jsonl  (训练数据)
2. qwen_finetune_rtx3090.py                        (RTX 3090 优化训练脚本)
3. autodl_start_rtx3090.sh                        (RTX 3090 优化启动脚本)
4. qwen_inference_test.py                          (推理测试脚本)
```

### 第二步: 修改数据路径

如果你的数据文件名不同，编辑 `autodl_start_rtx3090.sh`:

```bash
# 修改这一行
DATASET_PATH="/root/autodl-tmp/你的数据文件名.jsonl"
```

### 第三步: 运行训练

```bash
# 添加执行权限
chmod +x autodl_start_rtx3090.sh

# 开始训练
./autodl_start_rtx3090.sh
```

### 第四步: 监控训练

训练期间可以查看:

1. **训练日志**: 控制台输出
2. **TensorBoard**: 访问 http://localhost:6006
3. **GPU 监控**: `nvidia-smi`

### 第五步: 验证模型

训练完成后运行推理测试:

```bash
python qwen_inference_test.py --model_dir /root/autodl-tmp/qwen_trading_lora
```

## 常见问题

### Q: 显存不足怎么办？

**A**: 可以进一步调整:
- max_seq_length: 1536 → 1024
- lora_r: 32 → 16
- batch_size: 2 → 1 (gradient_accumulation: 16 → 32)

### Q: 训练太慢怎么办？

**A**: RTX 3090 训练时间是合理的。如果想更快:
- 考虑升级到 A100 或 A10
- 或者接受 RTX 3090 的性价比

### Q: 可以用更大的 batch_size 吗？

**A**: 不建议，RTX 3090 的 24GB 是瓶颈。强行增大可能导致 OOM。

### Q: LoRA Rank 32 会不会太小？

**A**: 不会，对于交易信号这种任务，32 足够了。如果需要更好的性能，可以后续微调更大的 Rank。

## 总结

✅ **RTX 3090 24GB 完全可以训练 Qwen3.5-7B**

关键优化点:
1. 降低序列长度 (2048 → 1536)
2. 降低 LoRA Rank (64 → 32)
3. 减小 batch size (4 → 2)
4. 增加梯度累积 (8 → 16)
5. 微调学习率 (1e-4 → 8e-5)
6. 增加训练步数 (1000 → 1500)

训练质量不会明显下降，但成本显著降低！
