# 订单簿与流动性

## 一、市场微结构（Market Microstructure）

很多顶级量化机构（例如 Citadel Securities、Jump Trading）的核心优势都来自这里。

### 核心思想

```
价格不是随机走的
而是由订单簿结构决定的
```

---

## 二、什么是市场微结构

**市场微结构**研究的是：

| 研究对象 | 说明 |
|---------|------|
| 订单 | Order |
| 成交 | Trade |
| 流动性 | Liquidity |
| 价格形成 | Price Formation |

传统金融研究：**价格走势**

但微结构研究：**订单如何推动价格**

## 三、订单簿（Order Book）

订单簿通常包含：

```
Bid side                Ask side

Price   Volume         Price   Volume
100     5 BTC          101     3 BTC
99      8 BTC          102     6 BTC
98      10 BTC         103     7 BTC
```

### 关键价格

| 价格类型 | 说明 |
|---------|------|
| Best Bid | 最优买价 |
| Best Ask | 最优卖价 |
| Mid Price | 中间价 = (bid + ask) / 2 |

---

## 四、订单簿 Alpha：Order Book Imbalance

**最经典 Alpha：订单簿不平衡**

### 公式

```
imbalance = (bid_volume - ask_volume) / (bid_volume + ask_volume)
```

### Python 示例

```python
def orderbook_imbalance(bid_vol, ask_vol):
    """订单簿不平衡"""
    return (bid_vol - ask_vol) / (bid_vol + ask_vol + 1e-6)
```

### 解释

| 指标 | 含义 | 预示 |
|------|------|------|
| imbalance > 0 | 买盘更强 | 价格上涨 |
| imbalance < 0 | 卖盘更强 | 价格下跌 |

## 五、订单簿深度

订单簿通常有：
- Level 1
- Level 2
- Level 10

例如：**Top 10 levels**

### 计算

```python
def bid_depth(bid_volumes):
    """买方深度"""
    return sum(bid_volumes[:10])

def ask_depth(ask_volumes):
    """卖方深度"""
    return sum(ask_volumes[:10])

def depth_ratio(bid_depth, ask_depth):
    """深度比"""
    return bid_depth / (ask_depth + 1e-6)
```

## 六、流动性指标

流动性非常重要。

### 典型指标

| 指标 | 说明 |
|------|------|
| spread | 买卖价差 |
| depth | 深度 |
| order flow | 订单流 |

---

### 1. Spread（买卖价差）

```python
def spread(best_bid, best_ask):
    """绝对价差"""
    return best_ask - best_bid

def spread_pct(best_bid, best_ask):
    """相对价差"""
    mid = (best_bid + best_ask) / 2
    return (best_ask - best_bid) / mid
```

**如果 spread 变大，说明：流动性下降**

---

### 2. 深度指标

```python
def market_liquidity_score(bid_depth, ask_depth, spread):
    """综合流动性评分"""
    total_depth = bid_depth + ask_depth
    return total_depth / (spread + 1e-6)
```

## 七、订单流（Order Flow）

订单流表示：**买单 vs 卖单**

### 指标：Order Flow Imbalance

```python
def order_flow_imbalance(buy_volume, sell_volume):
    """OFI - 订单流不平衡"""
    return buy_volume - sell_volume

def ofi_normalized(buy_volume, sell_volume):
    """标准化 OFI"""
    total = buy_volume + sell_volume
    return (buy_volume - sell_volume) / (total + 1e-6)
```

**如果 ofi > 0，说明：买方压力大**

## 八、交易冲击（Market Impact）

大订单会影响价格。

### 简单模型

```
impact ≈ order_size / market_volume
```

### 示例

| 参数 | 值 |
|------|---|
| 订单 | 100 BTC |
| 市场成交 | 1000 BTC |

**冲击：10%**

## 九、价格冲击模型

### 常见模型：Square Root Model

```
impact ≈ σ × √(order_size / volume)
```

**解释：**
- 订单越大
- 价格冲击越大

## 十一、订单簿 Alpha 策略示例

### 一个简单策略

```
imbalance > 0.3  → 做多
imbalance < -0.3 → 做空
```

### Python 示例

```python
def simple_orderbook_strategy(imbalance):
    """简单订单簿策略"""
    if imbalance > 0.3:
        return "BUY"
    elif imbalance < -0.3:
        return "SELL"
    else:
        return "HOLD"
```

## 十三、做市风险

**做市最大风险：库存风险**

例如：
- 持有大量 BTC
- 价格下跌

### 解决方法：Inventory Control

```python
def adjust_for_inventory(inventory, threshold):
    """根据库存调整报价"""
    if inventory > threshold:
        # 库存过大，减少买价
        return "reduce_bid"
    elif inventory < -threshold:
        # 负库存过大，减少卖价
        return "reduce_ask"
    else:
        return "normal"
```
