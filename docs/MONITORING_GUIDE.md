# 实时模拟交易监控指南

## 概述
本指南说明如何监控正在运行的实时模拟交易系统。

## 快速开始

### 1. 查看当前市场状态
```bash
python live_view.py
```
显示内容：
- 当前价格
- MA10/MA25均线
- 趋势方向
- 当前信号
- 最近10个信号

### 2. 每日检查
```bash
python daily_check.py
```
执行完整的每日检查：
- 市场状态
- 模拟结果
- 运行状态

### 3. 查看任务列表
在Claude Code中使用：
```
/tasks
```
查看所有正在运行的任务和后台进程。

### 4. 查看任务输出
```
/TaskOutput <task_id>
```
例如：`/TaskOutput b3tmy503y`

## 监控内容

### 实时指标
- **价格**: BTC/USDT当前价格
- **趋势**: 看涨(BULLISH)/看跌(BEARISH)/中性(NEUTRAL)
- **信号**: 买入(BUY)/卖出(SELL)/持有(HOLD)
- **均线**: MA10和MA25的当前值

### 性能指标
- **收益率**: 初始资金到最终价值的变化百分比
- **最大回撤**: 从最高点到最低点的最大跌幅
- **胜率**: 盈利交易占总交易的比例
- **风险收益比**: 平均盈利/平均亏损

### 交易记录
- **总交易次数**: 买入和卖出的总次数
- **最近交易**: 最后10笔交易的详细信息
- **持仓**: 当前持有的BTC数量
- **现金**: 当前可用现金

## 2周监控计划

### 第1天 - 第7天
- 每日运行 `daily_check.py`
- 记录市场观察
- 每周一进行周度评估

### 第8天 - 第14天
- 继续每日监控
- 对比历史回测与实时表现
- 评估参数调整需求
- 准备实盘计划

## 风险控制检查清单

### 每日检查
- [ ] 当前价格是否正常
- [ ] 趋势方向是否明确
- [ ] 信号是否合理
- [ ] 持仓是否在限制范围内
- [ ] 现金余额是否充足

### 每周检查
- [ ] 总收益率是否为正
- [ ] 最大回撤是否 < 10%
- [ ] 风险收益比是否 > 1.5
- [ ] 交易频率是否合理
- [ ] 胜率是否可接受

## 实盘准备 (2周后)

### 评估标准
1. **正收益率**: 模拟期间总收益 > 5%
2. **低回撤**: 最大回撤 < 10%
3. **稳定表现**: 每日收益波动较小
4. **信号一致性**: 实时信号与历史回测一致

### 实盘建议
- **初始仓位**: 5-10% 总资金
- **单笔仓位**: 最大 2-3%
- **止损**: -3% (硬性止损)
- **止盈**: +6-9% (2:1 风险收益比)
- **每日亏损限制**: -2% (强制平仓)

## 故障排除

### 模拟交易没有运行
1. 检查 `/tasks` 列表
2. 重新运行 `python start_live.py`
3. 查看任务输出获取错误信息

### 数据库连接错误
1. 检查 `.env` 文件配置
2. 验证数据库服务是否运行
3. 确认数据库表已创建

### 没有数据或数据过时
1. 运行 `npm run fetch-db` 获取新数据
2. 检查数据更新时间
3. 验证数据库连接
