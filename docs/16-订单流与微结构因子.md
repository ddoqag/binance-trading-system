# 订单流与微结构因子

## 一、市场微结构基础

**市场微结构**研究的是：
- 订单
- 成交
- 流动性
- 价格形成

传统金融研究：**价格走势**

但微结构研究：**订单如何推动价格**

---

**核心思想：**
```
价格不是随机走的
而是由订单簿结构决定的
```

---

## 二、订单簿（Order Book）

订单簿通常包含：

```
Bid side                Ask side

Price   Volume         Price   Volume
100     5 BTC          101     3 BTC
99      8 BTC          102     6 BTC
98      10 BTC         103     7 BTC
```

### 关键概念

| 概念 | 说明 |
|------|------|
| Best Bid | 最优买价 |
| Best Ask | 最优卖价 |
| Mid Price | 中间价 = (bid + ask) / 2 |

---

## 三、订单簿 Alpha：Order Book Imbalance

**最经典 Alpha：订单簿不平衡**

### 公式

```
imbalance = (bid_volume - ask_volume) / (bid_volume + ask_volume)
```

### Python 示例

```python
def orderbook_imbalance(bid_vol, ask_vol):
    """订单簿不平衡"""
    imbalance = (bid_vol - ask_vol) / (bid_vol + ask_vol + 1e-6)
    return imbalance
```

### 解释

- `imbalance > 0`：买盘更强，可能意味着价格上涨
- `imbalance < 0`：卖盘更强，可能意味着价格下跌

## 四、订单簿深度

订单簿通常有：
- Level 1
- Level 2
- Level 10

### 计算

```python
def bid_depth(bid_volumes, levels=10):
    """买方深度"""
    return sum(bid_volumes[:levels])

def ask_depth(ask_volumes, levels=10):
    """卖方深度"""
    return sum(ask_volumes[:levels])

def depth_ratio(bid_depth, ask_depth):
    """深度比"""
    return bid_depth / (ask_depth + 1e-6)
```

## 五、流动性指标

流动性非常重要。

### 典型指标

| 指标 | 说明 |
|------|------|
| spread | 买卖价差 |
| depth | 深度 |
| order flow | 订单流 |

---

### 1. Spread（买卖价差）

```python
def spread(best_bid, best_ask):
    """买卖价差"""
    return best_ask - best_bid

def spread_pct(best_bid, best_ask):
    """相对价差"""
    mid = (best_bid + best_ask) / 2
    return (best_ask - best_bid) / mid
```

**如果 spread 变大，说明：流动性下降**

---

### 2. 深度指标

```python
def market_depth(bid_sizes, ask_sizes, levels=5):
    """市场深度"""
    bid_depth = sum(bid_sizes[:levels])
    ask_depth = sum(ask_sizes[:levels])
    return bid_depth, ask_depth

def liquidity_score(bid_depth, ask_depth, spread):
    """综合流动性评分"""
    depth = bid_depth + ask_depth
    return depth / (spread + 1e-6)
```

## 六、订单流（Order Flow）

订单流表示：**买单 vs 卖单**

### 指标：Order Flow Imbalance

```python
def order_flow_imbalance(buy_volume, sell_volume):
    """订单流不平衡"""
    return buy_volume - sell_volume

def ofi_normalized(buy_volume, sell_volume):
    """标准化 OFI"""
    total = buy_volume + sell_volume
    return (buy_volume - sell_volume) / (total + 1e-6)
```

**如果 ofi > 0，说明：买方压力大**

---

### Trade Sign（交易方向标记）

```python
def trade_sign(price, mid_price):
    """标记交易方向"""
    if price > mid_price:
        return 1  # Buy
    elif price < mid_price:
        return -1  # Sell
    else:
        return 0  # Unknown
```

## 七、交易冲击（Market Impact）

大订单会影响价格。

### 简单模型

```
impact ≈ order_size / market_volume
```

### 示例

- 订单 = 100 BTC
- 市场成交 = 1000 BTC

**冲击：10%**

---

## 八、价格冲击模型

### 常见模型：Square Root Model

```
impact ≈ σ × √(order_size / volume)
```

**解释：**
- 订单越大
- 价格冲击越大

可以预测：**短期价格**

### 特征

- depth

### 模型

- Logistic Regression

### 示例

```python
def prepare_orderbook_features(df):
    """准备订单簿特征"""
    features = pd.DataFrame(index=df.index)

    # 不平衡特征
    features["imbalance"] = df["imbalance"]
    features["imbalance_ma5"] = df["imbalance"].rolling(5).mean()

    # 价差特征
    features["spread"] = df["spread"]
    features["spread_ma20"] = df["spread"].rolling(20).mean()

    # 深度特征
    features["depth_ratio"] = df["depth_ratio"]
    features["depth_ratio_chg"] = df["depth_ratio"].pct_change()

    # 订单流
    features["ofi"] = df["ofi"]
    features["ofi_cum"] = df["ofi"].rolling(10).sum()

    return features

def predict_price_movement(features, target):
    """训练价格预测模型"""
    from sklearn.ensemble import RandomForestClassifier

    model = RandomForestClassifier(n_estimators=100)
    model.fit(features, target)
    return model
```

## 十、订单簿 Alpha 策略示例

### 一个简单策略

```
imbalance > 0.3  → 做多
imbalance < -0.3 → 做空
```

### Python 示例

```python
def simple_ob_strategy(imbalance, threshold=0.3):
    """简单订单簿策略"""
    if imbalance > threshold:
        return "BUY"
    elif imbalance < -threshold:
        return "SELL"
    else:
        return "HOLD"
```

---

### 更复杂的策略

```python
def advanced_ob_strategy(imbalance, spread, depth_ratio):
    """高级订单簿策略"""

    # 信号1: 强不平衡
    strong_buy = imbalance > 0.3 and spread < spread.quantile(0.2)
    strong_sell = imbalance < -0.3 and spread < spread.quantile(0.2)

    # 信号2: 深度确认
    depth_buy = depth_ratio > 1.5
    depth_sell = depth_ratio < 0.5

    if strong_buy and depth_buy:
        return "BUY"
    elif strong_sell and depth_sell:
        return "SELL"
    else:
        return "HOLD"
```

## 十一、做市策略（Market Making）

做市策略非常常见。

### 核心

同时挂买单和卖单，赚取 bid-ask spread。

### 结构

```
Bid     Ask
100     101
```

**利润：1 USD**

---

### 做市风险

**做市最大风险：库存风险**

例如：
- 持有大量 BTC
- 价格下跌

### 解决方法：Inventory Control

```python
def adjust_quotes(inventory, mid_price, gamma=0.1, variance=0.01):
    """Avellaneda-Stoikov 模型"""
    reservation_price = mid_price - inventory * gamma * variance
    return reservation_price

def update_bid_ask(reservation_price, spread):
    """更新报价"""
    bid = reservation_price - spread / 2
    ask = reservation_price + spread / 2
    return bid, ask
```

---

## 十二、Avellaneda-Stoikov 模型

**经典做市模型**

核心思想：**根据库存动态调整报价**

### 报价公式

```
reservation_price = midprice - inventory × gamma × variance
```
