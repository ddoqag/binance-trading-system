# 五、行情采集服务

`market-data-service/binance_ws.py`

```python
import json
import websocket
from kafka import KafkaProducer

producer = KafkaProducer(
    bootstrap_servers="kafka:9092",
    value_serializer=lambda v: json.dumps(v).encode()
)

def on_message(ws, message):

    data = json.loads(message)

    tick = {
        "symbol": data["s"],
        "price": float(data["p"]),
        "volume": float(data["q"])
    }

    producer.send("market_ticks", tick)

ws = websocket.WebSocketApp(
    "wss://stream.binance.com:9443/ws/btcusdt@trade",
    on_message=on_message
)

ws.run_forever()
```

数据流：

```

   ↓
Kafka topic: market_ticks
```

# 三、构建因子计算框架

推荐结构：

```
factor-engine/

factors/
    momentum.py
    volatility.py
    volume.py
    orderflow.py

engine.py
```

核心执行器：

```python
import pandas as pd
from factors import momentum, volatility

class FactorEngine:

    def __init__(self):
        self.factors = {
            "momentum": momentum.momentum,
            "volatility": volatility.volatility
        }

    def compute(self, df):

        result = pd.DataFrame(index=df.index)

        for name, func in self.factors.items():

            result[name] = func(df)

        return result
```

输出：

```
time | momentum | volatility | volume_spike
```

# 四、因子 IC 测试（最重要）

IC = **Information Coefficient**

衡量：

```
因子 vs 未来收益
```

---

计算方法：

```python
def compute_ic(factor, future_return):

    return factor.corr(future_return)
```

完整示例：

```python
df["future_return"] = df["close"].pct_change(10).shift(-10)

ic = compute_ic(
    df["momentum"],
    df["future_return"]
)

print("IC:", ic)
```

IC解释：

| IC    | 含义 |
| ----- | -- |
| >0.1  | 很强 |
| 0.05  | 有效 |
| <0.02 | 噪音 |

```python
def rolling_ic(factor, future_return):
```

指标：


# 六、RL交易环境（Gym）

建议使用 **Gym 环境**。

示例：

```python
import gym
import numpy as np

class TradingEnv(gym.Env):

    def __init__(self, df):

        self.df = df
        self.step_id = 0
        self.position = 0

    def reset(self):

        self.step_id = 0
        self.position = 0

        return self.get_state()

    def step(self, action):

        price = self.df["close"].iloc[self.step_id]

        pnl = self.position * (price - self.prev_price)

        self.position = action

        reward = pnl

        self.step_id += 1

        done = self.step_id >= len(self.df) - 1

        return self.get_state(), reward, done, {}

    def get_state(self):

        row = self.df.iloc[self.step_id]

        return np.array([
            row["momentum"],
            row["volatility"],
            row["volume"]
        ])
```

# 三、Alpha Generator（自动生成因子）

系统通过组合算子生成新因子。

例如：

```
alpha1 = rank(close - mean(close,20))

alpha2 = momentum * volume_spike

alpha3 = zscore(volatility)
```

---

### Python 简化生成器

```python
import random

operators = ["+", "-", "*", "/"]

features = [
    "momentum",
    "volatility",
    "volume_spike"
]

def generate_alpha():

    f1 = random.choice(features)
    f2 = random.choice(features)

    op = random.choice(operators)

    return f"{f1} {op} {f2}"
```

生成：

```
momentum * volatility
volume_spike - momentum
```

```python
def momentum_strategy(df):

    ma20 = df["close"].rolling(20).mean()

    if ma20.iloc[-1] > ma50.iloc[-1]:
```

策略服务 gRPC：

```python
class StrategyService(trading_pb2_grpc.StrategyServiceServicer):

        if request.price > 50000:
            action = "SELL"
        else:
            action = "BUY"

        return trading_pb2.Signal(

            action=action,
            size=0.01
        )
```

下面是 **真正常见且长期有效的因子类别**（特别适用于加密市场）。

---

# 一、Momentum Alpha（动量因子）

最经典、最稳定的 Alpha。

核心思想：

```text
涨得多 → 继续涨
跌得多 → 继续跌
```

公式：

```python
momentum = close / close.shift(20) - 1
```

解释：

```text
过去20天涨幅
```

优点：

* 简单
* 在大多数市场有效

缺点：

* 在震荡市场容易失效

---

# 二、Mean Reversion Alpha（均值回归）

核心思想：

```text
价格偏离均值 → 回归均值
```

常见公式：

```python
zscore = (price - mean) / std
```

策略逻辑：

```text
zscore > 2  → 做空
zscore < -2 → 做多
```

在 **高频和做市策略**中非常常见。

---

# 三、Volatility Alpha（波动率因子）

波动率变化常常领先价格变化。

公式：

```python
volatility = returns.rolling(30).std()
```

常见信号：

```text
波动率上升 → 市场即将突破
波动率下降 → 市场进入盘整
```

---

# 四、Volume Alpha（成交量因子）

成交量经常提前反映资金流。

例子：

```python
volume_spike = volume / volume.rolling(20).mean()
```

如果：

```text
volume_spike > 3
```

说明：

```text
有异常资金进入
```

---

# 五、Order Flow Alpha（订单流）

这是 **高频交易最核心的 Alpha**。

来源：

```text
订单簿变化
买卖盘压力
```

示例：

```python
imbalance = (bid_volume - ask_volume) / (bid_volume + ask_volume)
```

如果：

```text
imbalance > 0.3
```

说明：

```text
买盘更强
```

---

# 六、Liquidity Alpha（流动性因子）

示例：

```python
spread = ask_price - bid_price
```

如果：

```text

```

可能意味着：

```text

```

# 七、Market Microstructure Alpha（微观结构）

来自交易行为。

常见信号：

```text
订单撤单
隐藏订单
撮合频率
```

例如：


# 一、Look-Ahead Bias（未来函数）

最常见错误。

定义：

```text
使用了未来数据
```

例如：

```python
df["future_return"] = df["close"].shift(-1)

signal = factor > 0
pnl = signal * df["future_return"]
```

如果你在计算因子时 **误用了未来数据**，回测会极度乐观。

典型错误：

```python
rolling_mean = close.rolling(20).mean()
signal = close > rolling_mean
```

如果 rolling 计算不小心使用未来数据，就会产生 bias。

解决方案：

```text
所有因子必须只使用 t 时刻之前数据
```

严格规范：

```python
factor = factor.shift(1)
```

---

# 二、Survivorship Bias（幸存者偏差）

如果只用 **当前存在的资产** 做回测，就会产生巨大偏差。

例如：

```text
只回测当前 Top100 币
```

问题：

```text
很多已经退市或归零的币被忽略
```

真实市场：

```text
大量项目归零
```

解决方法：

```text
使用历史完整资产池
```

例如：

```
2020年存在的所有币
```

而不是现在的币。

---

# 三、Data Snooping Bias（数据挖掘偏差）

定义：

```text
在同一数据上反复测试策略
```

例如：

```
测试1000个因子
```

一定会出现：

```text
几个看起来很好的因子
```

但其实只是：

```text
随机噪声
```

解决方法：

```text
训练集 / 测试集
```

例如：

```
2018-2022 → 训练
2023-2024 → 测试
```

# 四、Overfitting（过拟合）

解决方案：


或者：


# 九、Execution Bias（执行偏差）

# 五、随机表达式生成

# 四、库存控制

自动平衡仓位。
