# HFT 项目 - 需求规格说明书 (SRS)

> Software Requirements Specification
> 版本: 1.0
> 最后更新: 2026-03-30

---

## 1. 引言

### 1.1 目的

本文档描述HFT延迟队列RL系统的软件需求规格。

### 1.2 范围

系统包含以下主要功能模块：
- Go执行引擎（订单簿、撮合、延迟模拟）
- Python AI层（SAC Agent、特征工程）
- 共享内存通信（mmap、序列锁）
- 币安数据接入（WebSocket）

### 1.3 定义和缩写

| 缩写 | 全称 | 说明 |
|------|------|------|
| HFT | High-Frequency Trading | 高频交易 |
| RL | Reinforcement Learning | 强化学习 |
| SAC | Soft Actor-Critic | 一种RL算法 |
| OFI | Order Flow Imbalance | 订单流不平衡 |
| VPIN | Volume-synchronized Probability of Informed Trading | 毒性流量指标 |
| mmap | Memory-mapped file | 内存映射文件 |

---

## 2. 总体描述

### 2.1 系统视角

系统采用Go + Python异构架构，通过共享内存实现微秒级通信。

### 2.2 用户特征

- 量化交易员：使用系统进行自动化交易
- 系统管理员：监控和维护系统运行

### 2.3 运行环境

- **硬件**: x86_64服务器，建议靠近币安服务器
- **操作系统**: Linux (Ubuntu 22.04+)
- **网络**: 低延迟网络连接
- **依赖**: Go 1.21+, Python 3.10+, PyTorch 2.0+

## 3. 功能需求

### 3.1 执行引擎 (Go)

#### FR-001: 订单簿维护
- **优先级**: P0
- **描述**: 系统应能维护L2级别订单簿
- **验收标准**:
  - 支持至少50档深度
  - 更新延迟 < 100μs
  - 零拷贝实现

#### FR-002: 撮合引擎
- **优先级**: P0
- **描述**: 系统应能模拟交易所撮合逻辑
- **验收标准**:
  - FIFO队列实现
  - 支持隐藏流动性
  - 撮合延迟 < 50μs

#### FR-003: 延迟引擎
- **优先级**: P1
- **描述**: 系统应能模拟真实网络延迟
- **验收标准**:
  - 支持正态分布延迟
  - 可配置base latency和jitter

### 3.2 AI决策层 (Python)

#### FR-004: SAC Agent
- **优先级**: P0
- **描述**: 系统应实现SAC算法进行交易决策
- **验收标准**:
  - Twin Critics架构
  - 自动温度调节
  - 支持连续动作空间

#### FR-005: 特征工程
- **优先级**: P0
- **描述**: 系统应计算订单流特征
- **验收标准**:
  - OFI计算
  - 微观价格计算
  - 队列位置跟踪

#### FR-006: Meta-Agent (v4.0)
- **优先级**: P1
- **描述**: 系统应实现元调度器
- **验收标准**:
  - 支持多策略切换
  - 市场状态检测

### 3.3 通信层

#### FR-007: 共享内存通信
- **优先级**: P0
- **描述**: 系统应通过mmap实现Go-Python通信
- **验收标准**:
  - 延迟 < 1μs
  - 无锁同步（序列锁）
  - 缓存行对齐

#### FR-008: 币安WebSocket
- **优先级**: P0
- **描述**: 系统应能接入币安实时数据
- **验收标准**:
  - L2深度数据流
  - 实时成交数据流
  - 自动重连机制

### 3.4 风控系统

#### FR-009: 基础风控
- **优先级**: P0
- **描述**: 系统应实现基础风险控制
- **验收标准**:
  - 仓位限制检查
  - 日亏损熔断
  - 自成交防护

#### FR-010: SAC安全层
- **优先级**: P1
- **描述**: 系统应实现RL动作的安全检查
- **验收标准**:
  - VPIN超标检测
  - 滑点异常检测
  - 动态降级

### 3.5 实盘交易 (v3.0)

#### FR-011: 实盘下单
- **优先级**: P0
- **描述**: 系统应支持币安实盘交易
- **验收标准**:
  - 限价单/市价单
  - 订单状态跟踪
  - API限速管理

#### FR-012: 订单管理
- **优先级**: P0
- **描述**: 系统应实现完整订单生命周期管理
- **验收标准**:
  - 状态机实现
  - WAL日志
  - 错误重试
