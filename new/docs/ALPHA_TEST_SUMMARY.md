## 失败原因分析

### 1. 市场效率

在秒级时间尺度上，币安BTC/USDT市场接近弱式有效:
- 公开信息（订单流、成交数据）已被快速定价
- 任何可预测的模式会在毫秒级内被套利消除
- 高频做市商已捕获所有可预测的利润

### 2. 信号质量

| 信号 | 问题 |
|------|------|
| OFI | 与价格变化相关性极弱 (~0.05) |
| Microprice | 均值回归过快，无法用于方向预测 |
| Trade Imbalance | 被大单操纵，信噪比过低 |
| Intensity | 与波动率混淆，非方向性指标 |

### 3. 交易成本

即使信号有微弱优势，也被交易成本抹去:
- Maker Fee: 0.02%
- Taker Fee: 0.05%
- 滑点: 0.01-0.03%
- **总成本**: ~0.08% 每轮交易

需要 >52% 的准确率才能盈亏平衡。

## 战略转向

### 从预测到做市

| 维度 | 预测型策略 | 做市型策略 |
|------|-----------|-----------|
| 核心假设 | 价格可预测 | 流动性需求持续 |
| 盈利模式 | 方向性Alpha | 点差 + 返佣 |
| 竞争优势 | 信息优势 | 速度优势 |
| 风险来源 | 预测错误 | 库存风险 + 毒流 |
| 可持续性 | 低（模式衰减快） | 高（流动性需求稳定） |

### 新策略核心

1. **队列位置优化** - 永远在队列前30%
2. **毒流检测** - 马氏距离 > 0.3 时停止做市
3. **点差捕获** - Spread ≥ 2 ticks 时挂被动单

### 预期收益来源

| 来源 | 预期贡献 | 确定性 |
|------|---------|--------|
| Maker Rebate | 0.02% | 高 |
| Spread Capture | 1-5 bps | 中 |
| 队列优先 | 减少机会成本 | 高 |
| 毒流防御 | 避免亏损 | 高 |
