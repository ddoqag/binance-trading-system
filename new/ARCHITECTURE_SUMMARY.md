# HFT Engine 架构总结

## 一、入口流程

```
┌─────────────────────────────────────────────────────────────┐
│                        入口 (main.go)                        │
│  - 参数解析: symbol, paper/live trading, margin mode         │
│  - 创建 HFTEngine                                            │
│  - 启动引擎 → 等待信号 → 优雅关闭                             │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│              NewHFTEngine (引擎初始化)                        │
├─────────────────────────────────────────────────────────────┤
│  1. 初始化共享内存 (SHMManager)                              │
│  2. 初始化 WebSocket 管理器 (WebSocketManager)               │
│  3. 初始化订单执行器 (OrderExecutor/MarginExecutor)          │
│  4. 初始化风险管理器 (RiskManager)                            │
│  5. 初始化 WAL (Write-Ahead Log)                             │
│  6. 初始化降级管理器 (EnhancedDegradeManager)                │
│  7. 初始化防御系统 (DefenseManager)                          │
│  8. 初始化指标收集器 (MetricsCollector)                      │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                  Start (引擎启动)                            │
├─────────────────────────────────────────────────────────────┤
│  1. 设置 WebSocket 处理器 (深度/成交)                        │
│  2. 连接币安 WebSocket                                       │
│  3. 启动三大循环:                                            │
│     - decisionLoop   (决策执行, 100ms 心跳)                  │
│     - marketDataLoop (市场数据刷新, 50ms)                    │
│     - monitorLoop    (监控日志, 5s)                          │
└─────────────────────────────────────────────────────────────┘
```

## 二、核心循环逻辑

### 2.1 决策循环 (decisionLoop → processDecision)

```
┌──────────────────────────────────────────────────────────────────────┐
│                        processDecision 流程                           │
├──────────────────────────────────────────────────────────────────────┤
│  1. 从共享内存读取 AI 决策 (action, size, price, confidence)          │
│                                                                       │
│  2. 降级检查 (DegradeManager)                                         │
│     └── CanTrade(isClosing) → 根据降级级别决定是否允许交易             │
│                                                                       │
│  3. 仓位调整 (GetMaxPositionSize)                                     │
│     └── 根据降级级别动态调整最大仓位                                   │
│                                                                       │
│  4. 风险检查 (RiskManager)                                            │
│     └── CanExecute(action, size, inventory)                           │
│         - 检查仓位限制                                                │
│         - 检查每日亏损限制                                            │
│         - 检查最大回撤                                                │
│                                                                       │
│  5. 防御检查 (DefenseManager) - 毒流防护                              │
│     └── CanPlaceOrder(side, size)                                     │
│         - TOXIC 模式: 只开放安全侧 (买压大→只卖单, 卖压大→只买单)       │
│                                                                       │
│  6. 执行决策                                                          │
│     ├── executeMarginDecision (杠杆模式)                              │
│     └── executeSpotDecision (现货模式)                                │
│                                                                       │
│  7. 更新状态                                                          │
│     ├── AcknowledgeDecision (确认决策)                                │
│     ├── Update inventory (更新持仓)                                   │
│     └── Log execution (记录执行日志)                                  │
└──────────────────────────────────────────────────────────────────────┘
```

### 2.2 市场数据循环 (marketDataLoop)

```
┌──────────────────────────────────────────────────────────────────────┐
│                    onDepthUpdate (深度更新)                           │
├──────────────────────────────────────────────────────────────────────┤
│  1. 接收币安深度数据 (bestBid, bestAsk, OFI)                          │
│                                                                       │
│  2. 计算微价格 (MicroPrice)                                           │
│                                                                       │
│  3. 防御系统注入 (DefenseManager)                                     │
│     ├── OnMarketTick (市场状态更新)                                   │
│     ├── UpdateMarketState (FSM 状态机更新)                            │
│     │   ├── decideMode (决定模式: NORMAL/DEFENSIVE/TOXIC)             │
│     │   ├── updatePolicyForMode (更新执行策略)                        │
│     │   └── adjustOrders (调整/取消订单)                              │
│                                                                       │
│  4. 写入共享内存 (Python AI 消费)                                     │
│     └── WriteMarketData(bid, ask, microPrice, OFI, ...)               │
└──────────────────────────────────────────────────────────────────────┘
```

## 五、核心模块职责

| 模块 | 文件 | 核心职责 |
|------|------|----------|
| **HFTEngine** | `engine.go` | 主协调器，初始化所有子系统，管理生命周期 |
| **WebSocketManager** | `websocket_manager.go` | 币安 WebSocket 连接管理，深度/成交数据处理 |
| **SHMManager** | `shm_manager.go` | 共享内存 IPC，与 Python AI 零拷贝通信 |
| **DefenseManager** | `defense_manager.go` | 毒流检测 + FSM 状态管理，三层防御 |
| **OrderDefenseFSM** | `order_defense_fsm.go` | 订单状态机，NORMAL/DEFENSIVE/TOXIC 模式 |
| **ToxicDetector** | `toxic_detector.go` | 毒流评分算法 (OFI + 爆发 + 波动率) |
| **RiskManager** | `risk_manager.go` | 仓位/亏损/回撤限制 |
| **EnhancedRiskManager** | `risk_enhanced.go` | 高级风控 (止损/止盈/追踪止损/持仓时间) |
| **EnhancedDegradeManager** | `degrade.go` | 系统降级 + 熔断器 |
| **WAL** | `wal.go` | 预写日志，灾难恢复 |
| **ModelManager** | `model_manager.go` | ONNX 模型热更新 + A/B 测试 |
| **OrderExecutor** | `order_executor.go` | 现货订单执行 |
| **MarginExecutor** | `margin_executor.go` | 杠杆交易执行 |

## 八、防御系统状态机

```
                    ┌─────────────┐
                    │   NORMAL    │
                    │  (正常模式)  │
                    └──────┬──────┘
                           │ toxic_score > 0.6
                           ↓
                    ┌─────────────┐
                    │  DEFENSIVE  │
                    │  (防御模式)  │ ◄───┐
                    └──────┬──────┘     │
                           │            │ conditions_normalized
              toxic_score > 0.8          │
                           │            │
                           ↓            │
                    ┌─────────────┐     │
           ┌───────│    TOXIC    │─────┘
           │       │  (毒流模式)  │
           │       └─────────────┘
           │              │
           │              │ post_toxic_cooldown (2s)
           │              │
           └──────────────┘
```

### 8.1 三种模式行为

| 模式 | 撤单率 | 订单超时 | 侧边控制 | 最大订单数 |
|------|--------|----------|----------|------------|
| NORMAL | 20% | 5s | 双边开放 | 无限制 |
| DEFENSIVE | 60% | 2s | 双边开放 | 减少 |
| TOXIC | 95% | 500ms | 只开安全侧 | 2 |

### 8.2 毒流评分公式

```
ToxicScore = 0.30 * |OFI| +
             0.25 * Burst +
             0.20 * Volatility +
             0.15 * LargeTradeRatio +
             0.10 * |FlowImbalance|

阈值:
- TOXIC:  > 0.8
- DEFENSIVE: > 0.6
```

## 九、共享内存协议

### 9.1 数据结构

```go
// MarketSnapshot - Go → Python
 type MarketSnapshot struct {
     TimestampNs        uint64
     BestBid            float64
     BestAsk            float64
     MicroPrice         float64
     OrderFlowImbalance float64
     TradeImbalance     float64
     BidQueuePosition   float64
     AskQueuePosition   float64
     AdverseScore       float64
     ToxicProbability   float64
     ...
 }

 // OrderCommand - Python → Go
 type OrderCommand struct {
     CommandID      uint64
     OrderType      uint32
     Side           uint32
     Price          float64
     Quantity       float64
     MaxSlippageBPS float64
     ...
 }

 // AIContext - Python → Go (MoE 权重)
 type AIContext struct {
     AIPosition       float64
     AIConfidence     float64
     MoEWeight0-3     float64
     RegimeCode       uint32
     NumActiveExperts uint32
 }
```

### 9.2 共享内存布局

```

│ Offset 0: SharedMemoryHeader (魔数/版本/大小)    │
├─────────────────────────────────────────────────┤
│ Offset 64: MarketSnapshot (市场数据)             │
├─────────────────────────────────────────────────┤
│ Offset 2048: OrderCommand (订单命令)             │
├─────────────────────────────────────────────────┤
│ Offset 4096: AIContext (AI 决策上下文)           │
├─────────────────────────────────────────────────┤
│ Offset 8192: OrderStatusUpdate[] (订单状态数组)  │
├─────────────────────────────────────────────────┤
│ Offset ...: TradeExecution[] (成交记录数组)      │
└─────────────────────────────────────────────────┘
```
